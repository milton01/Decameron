Versión 2.0.3

Fecha de Publicación: 20 de agosto de 2011

  • Seguridad

    • Se hizo una mejora en los drivers MySQL y MySQLi para evitar un vector potencial para la inyección SQL en sitios que usan conjuntos de caracteres mutibyte en la conexión del cliente de base de datos.

      Una incompatibilidad en versiones de PHP < 5.2.3 y de MySQL 5.0.7 con mysql_set_charset() crean una situación en la que los conjuntos de caracteres multibyte en esos entornos pueden exponerse potencialmente a un ataque de inyección SQL. Latin-1, UTF-8 y otros conjuntos de carateres de “ASCII bajo” no se ven afectados en esos entornos.

      Si está corriendo o considera correr un conjunto de caracteres multibye en su conexión, por favor preste atención al entorno del servidor con el que está desplegando para asegurarse que no es vulnerable.

  • Cambios Generales
    • Se corrigió un error que había en un comentario en el archivo index.php.
    • Se agregaron userdata a la Clase Session para la salida del perfilador. Además se agregó una bandera mostrar/ocultar en los Encabezados HTTP, Datos de Sesión y Variables de Configuración.
    • Se eliminó el uso interno de la constante EXT.

    • form_open() ahora sólo añade un campo de protección oculto (Cross-site Reference Forgery) cuando la acción del formulario es interna y se establece el método POST.
    • Las funciones plural() y singular() se modificaron en Inflector helper para apoyar mucho más palabras.
  • Bibliotecas
    • Se alteró Session para usar una mayor coincidencia en la cadena user_agent. Ver las notas de actualización de 2.0.2 a 2.0.3 si se usan sesiones de base de datos.
    • Se agregó is_unique a la Clase Form Validation.

    • Se cambió $this->db->having() para insertar comillas usando escape() en lugar de escape_str().
